## Repository Purpose

This is **ZengLiangYi's GitHub Profile README** repository (`ZengLiangYi/ZengLiangYi`). It is a special GitHub repository whose `README.md` is displayed on the user's GitHub profile page. The repo contains no application code — it is purely a profile showcase with automated content pipelines.

## Architecture

- **`README.md`** — The profile page, written in HTML/Markdown. Contains dynamic sections updated by CI (blog posts between `<!-- BLOG-POST-LIST:START -->` / `<!-- BLOG-POST-LIST:END -->` markers). Written in a mix of Chinese and English.
- **`docker-compose.yml`** — Spins up a local [RSSHub](https://docs.rsshub.app/) instance to convert Juejin (掘金) articles into RSS feeds. Only used inside GitHub Actions, not for local development.
- **`profile-summary-card-output/`** — Auto-generated SVG stat cards in multiple themes (radical, dracula, github_dark, etc.). Generated by CI and committed directly to `main`. Do not manually edit these files.
- **`output` branch** — Contains snake animation SVGs generated by CI. Referenced from README via raw GitHub URLs.

## GitHub Actions Workflows

All workflows support `workflow_dispatch` for manual triggering.

| Workflow | File | Schedule | Purpose |
|----------|------|----------|---------|
| Blog Posts | `.github/workflows/blog-post-workflow.yml` | Daily 00:00 UTC | Starts RSSHub via Docker Compose, fetches Juejin posts, updates README |
| Snake Animation | `.github/workflows/snake.yml` | Every 12h | Generates contribution graph snake SVG, pushes to `output` branch |
| Profile Summary Cards | `.github/workflows/profile-summary-cards.yml` | Every 12h | Generates GitHub stats SVGs into `profile-summary-card-output/` on `main` |
